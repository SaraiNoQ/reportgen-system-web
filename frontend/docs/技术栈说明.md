# 技术栈说明

## 核心框架

- Next.js App Router：路由、布局、服务端渲染和静态构建。
- React：页面组件与交互状态。
- TypeScript：领域类型、组件 props 和 adapter 边界。
- pnpm：依赖管理。

## 样式与 UI

- Tailwind CSS v4：原子样式与主题 token。
- CSS variables：统一颜色、字体、阴影和圆角。
- lucide-react：线性图标。
- clsx、tailwind-merge：条件 className 组合。

## 数据层

当前使用 `Core API + API adapter`：

- Core API 默认地址为 `http://127.0.0.1:8000/api/v1`。
- 后端开发期数据存放在仓库根目录 `data/*.json`。
- `src/lib/mock/data.ts` 仅作为后端不可用时的少量页面 fallback 和演示种子参考。
- `src/lib/types/domain.ts` 存放前端领域类型。
- `src/lib/services/api.ts` 暴露页面调用接口。

后续接入真实数据库、对象存储和 AI Worker 时，优先保持 `api.ts` 函数签名稳定，不直接改页面组件。

## 当前依赖脚本

```bash
pnpm dev
pnpm build
pnpm lint
pnpm typecheck
```

## 未来可替换点

- 文件上传：当前通过 Core API 保存文件元数据，后续替换为对象存储或后端二进制上传接口。
- OCR/解析：接入后端解析任务接口和轮询/SSE 状态更新。
- 大模型处理：由后端统一调用模型服务，前端仅展示建议、置信度和来源。
- 报告编辑器：当前为轻量 textarea 封装，可替换为 Tiptap/ProseMirror。
- Word 导出：当前通过 Core API 记录导出任务和文件名，后续替换为真实模板渲染和对象存储下载地址。
