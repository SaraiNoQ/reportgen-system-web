# 技术栈说明

## 核心框架

- Next.js App Router：路由、布局、服务端渲染和静态构建。
- React：页面组件与交互状态。
- TypeScript：领域类型、组件 props 和 adapter 边界。
- pnpm：依赖管理。

## 样式与 UI

- Tailwind CSS v4：原子样式与主题 token。
- CSS variables：统一颜色、字体、阴影和圆角。
- lucide-react：线性图标。
- clsx、tailwind-merge：条件 className 组合。

## 数据层

当前使用 `mock + API adapter`：

- `src/lib/mock/data.ts` 存放演示数据。
- `src/lib/types/domain.ts` 存放前端领域类型。
- `src/lib/services/api.ts` 暴露页面调用接口。

后续接入真实服务时，优先替换 `api.ts` 内部实现，不直接改页面组件。

## 当前依赖脚本

```bash
pnpm dev
pnpm build
pnpm lint
pnpm typecheck
```

## 未来可替换点

- 文件上传：从 mock 状态改为对象存储或后端上传接口。
- OCR/解析：接入后端解析任务接口和轮询/SSE 状态更新。
- 大模型处理：由后端统一调用模型服务，前端仅展示建议、置信度和来源。
- 报告编辑器：当前为轻量 textarea 封装，可替换为 Tiptap/ProseMirror。
- Word 导出：从 mock 按钮改为后端导出任务或模板渲染接口。
