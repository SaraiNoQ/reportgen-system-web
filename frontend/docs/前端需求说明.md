# 前端需求说明

## 背景

系统用于将机床检测过程中的原始记录文件输入前端工作台，由 Core API 承接项目、文件、字段、规则、报告、用户、消息和日志等 Web 业务能力。当前前端阶段以“可联调的业务闭环”为目标，页面通过统一 API adapter 调用后端，后端以根目录 `data/*.json` 作为开发期轻量数据层。

## 需求来源

- 主需求依据：`docs/需求规格说明书.md`。
- 视觉参考：`DESIGN.md`。
- 原型参考：Figma Make 初版页面仅作为早期结构参考；当原型与需求规格冲突时，以需求规格为准。

## 用户角色

- 管理员：管理用户、查看日志、维护项目与规则。
- 编制员/检测人员：上传原始记录、查看解析结果、生成并编辑报告。
- 审核员：在后续业务中查看报告与字段溯源；当前前端不提供独立“审核与流转”页面。

## 当前模块范围

### 登录

- 输入账号和密码。
- 支持记住账号、忘记密码入口展示。
- 调用 Core API 登录后进入 `/projects`；后端不可用时仅保留本地 fallback 便于页面演示。

### 账号与消息中心

- 展示当前登录用户、角色、部门、登录时间和当前项目。
- 展示系统消息，支持单条已读和全部已读。
- 展示可切换项目列表；当前项目选择会通过 Core API 写入用户偏好，并保留本地兜底。
- 支持退出登录，退出时清理本地 session 和当前项目兜底状态。

### 项目管理

- 展示项目列表、状态、进度和更新时间。
- 支持搜索和状态筛选。
- 提供继续处理入口，跳转到原始记录上传或报告生成页面。

### 原始记录上传与解析

- 支持 PDF、Word、Excel、图片等文件类型的上传 UI，并通过 Core API 写入文件元数据、解析事件和字段集合。
- 展示解析配置、文件状态、OCR 状态、解析状态、解析时间线。
- 展示字段提取结果与置信度，支持单字段编辑和全部字段批量保存。
- 支持新增文件、失败文件重试、人工补录字段、解析结果导出和生成报告跳转。

### 规则配置

- 展示模板分类树。
- 展示模板详情、字段定义、字段详情、版本管理。
- 展示规则编辑器、溯源关系、判定规则和结论模板入口。
- 支持通过 Core API 保存规则并记录模板版本。

### 报告生成

- 选择报告模板和检测项。
- 展示报告目录与正文编辑区。
- 支持正文编辑、保存草稿、应用智能建议、章节类别更新、目录排序、预览、导出 Word/PDF、上传更正版和版本回退的前后端联调。
- 当前系统不包含审核流程，报告生成后直接通过交付文件区预览或导出。

### 用户管理

- 展示用户列表、角色、部门、账号状态和最近登录时间。
- 提供新增用户、批量导入、导出、编辑和启停用操作入口。

### 日志管理

- 展示登录、上传、解析、规则配置、报告生成等操作日志。
- 支持关键词、模块、结果筛选、日志详情、日志导出和软删除项目恢复。

## 导航要求

- 用户进入系统后默认到“原始记录上传”。
- 左侧一级导航包含：原始记录上传、规则配置、报告生成。
- 项目管理、用户管理、日志管理放入左侧“管理”折叠菜单。
- 顶部账号入口进入 `/account`，用于消息处理、账号信息和当前项目切换。
- 不展示“驾驶舱”和“审核与流转”页面。

## 非目标

- 不实现真实二进制文件上传到对象存储。
- 不实现真实 OCR、LLM 调用和字段抽取；当前由 Core API 写入模拟解析状态、事件和字段。
- 不实现真实 Word/PDF 文件渲染；当前由 Core API 记录导出任务、文件名和交付记录。
- 不实现完整 RBAC 权限鉴权；当前已有本地 JWT 登录态、会话校验和用户状态校验。
- 不实现独立审核流转页面。
