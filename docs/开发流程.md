# 开发流程

## 1. 需求确认

开发前先阅读：

- `docs/需求规格说明书.md`
- `DESIGN.md`
- `docs/前端需求说明.md`
- `docs/前端开发规范SPEC.md`

当 Figma 初版原型与需求规格不一致时，以需求规格为准。

## 2. 页面开发

1. 明确页面所属模块和路由。
2. 在 `src/lib/types/domain.ts` 补齐领域类型。
3. 在 `src/lib/mock/data.ts` 增加演示数据。
4. 在 `src/lib/services/api.ts` 增加 adapter 方法。
5. 在 `src/app` 增加页面文件。
6. 如果有复杂交互，在 `src/components/pages` 增加 client component。
7. 优先复用 `src/components/ui` 和 `src/components/layout` 中的组件。

## 3. UI 调整

- 先改共享组件 token，再改页面局部样式。
- 保持字体大小稳定，优先通过 padding、gap、列宽、控件高度和圆角调整密度。
- 工作台页面不做营销型 hero，不使用大面积装饰渐变。
- 表格与编辑区以清晰、紧凑、可扫描为主。

## 4. 验证

每轮较大修改后执行：

```bash
pnpm lint
pnpm typecheck
pnpm build
```

前端视觉改动还应启动：

```bash
pnpm dev
```

并检查以下页面：

- `/login`
- `/records`
- `/rules`
- `/reports`
- `/projects`
- `/system/users`
- `/system/logs`

## 5. 接入真实后端

真实接口接入时推荐顺序：

1. 登录鉴权。
2. 项目列表。
3. 文件上传与解析任务。
4. 字段抽取结果与置信度。
5. 规则模板与版本。
6. 报告生成、保存、导出。
7. 用户管理和日志管理。

页面组件不应直接依赖具体 HTTP client，应继续通过 adapter 调用。
