# 前端需求说明

## 背景

系统用于将机床检测过程中的原始记录文件输入前端工作台，由后续服务完成 OCR、字段抽取、大模型处理、规则判定和报告生成。当前前端阶段以“可演示的业务闭环”为目标，通过 mock 数据模拟流程。

## 需求来源

- 主需求依据：`docs/需求规格说明书.md`。
- 视觉参考：`DESIGN.md`。
- 原型参考：Figma Make 初版页面仅作为早期结构参考；当原型与需求规格冲突时，以需求规格为准。

## 用户角色

- 管理员：管理用户、查看日志、维护项目与规则。
- 编制员/检测人员：上传原始记录、查看解析结果、生成并编辑报告。
- 审核员：在后续业务中查看报告与字段溯源；当前前端不提供独立“审核与流转”页面。

## 当前模块范围

### 登录

- 输入账号和密码。
- 支持记住账号、忘记密码入口展示。
- mock 登录后进入 `/records`。

### 项目管理

- 展示项目列表、状态、进度和更新时间。
- 支持搜索和状态筛选。
- 提供继续处理入口，跳转到原始记录上传或报告生成页面。

### 原始记录上传与解析

- 支持 PDF、Word、Excel、图片等文件类型的上传 UI。
- 展示解析配置、文件状态、OCR 状态、解析状态、解析时间线。
- 展示字段提取结果与置信度。
- 支持模拟新增文件、失败文件重试和生成报告跳转。

### 规则配置

- 展示模板分类树。
- 展示模板详情、字段定义、字段详情、版本管理。
- 展示规则编辑器、溯源关系、判定规则和结论模板入口。
- 支持保存规则的 mock 反馈。

### 报告生成

- 选择报告模板和检测项。
- 展示报告目录与正文编辑区。
- 支持正文编辑、保存草稿、应用智能建议、预览、导出 Word 和提交审核的前端交互。
- 当前提交审核仅更新页面提示和 mock 状态，不跳转到独立审核页面。

### 用户管理

- 展示用户列表、角色、部门、账号状态和最近登录时间。
- 提供新增用户、批量导入、导出、编辑和启停用操作入口。

### 日志管理

- 展示登录、上传、解析、规则配置、报告生成等操作日志。
- 支持关键词、模块、结果筛选 UI。

## 导航要求

- 用户进入系统后默认到“原始记录上传”。
- 左侧一级导航包含：原始记录上传、规则配置、报告生成。
- 项目管理、用户管理、日志管理放入左侧“管理”折叠菜单。
- 不展示“驾驶舱”和“审核与流转”页面。

## 非目标

- 不实现真实文件上传。
- 不实现真实 OCR、LLM 调用和字段抽取。
- 不实现真实 Word 导出。
- 不实现真实权限鉴权。
- 不实现独立审核流转页面。
