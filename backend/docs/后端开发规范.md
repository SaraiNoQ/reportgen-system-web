# 后端开发规范

## 分层规范

后端按以下层次组织代码：

```text
api -> schemas -> services -> repositories -> models/db
```

- `api` 只处理 HTTP 入参、鉴权依赖、状态码和响应模型。
- `schemas` 存放 Pydantic 输入输出结构。
- `services` 编排业务流程和事务边界。
- `repositories` 封装数据库读写。
- `models` 存放 SQLAlchemy ORM 模型。
- `tasks` 存放 Celery 任务入口和任务投递逻辑。
- `storage` 封装对象存储访问。

API 层不直接写 SQL，不直接访问对象存储，不直接调用大模型。

当前开发期使用 `app/services/mock_store.py` 读写根目录 `data/*.json`，可暂不经过 `repositories` 和 ORM；新增业务集合仍必须同步 schema、store、API、测试和 `docs/Core API 接口说明.md`。后续迁移 PostgreSQL 时再把 JSON 文件读写替换为 repository/ORM 实现。

## 命名规范

- 路由文件使用业务复数名，例如 `users.py`、`projects.py`、`parse_jobs.py`。
- 数据库表名使用小写复数或业务前缀，例如 `auth_users`、`parse_jobs`。
- Pydantic schema 使用明确后缀：
  - `UserCreate`
  - `UserUpdate`
  - `UserRead`
  - `ParseJobRead`
- Service 使用单数业务名，例如 `UserService`、`ParseJobService`。

## 接口规范

- 所有前端接口统一挂载在 `/api/v1`。
- 大数据量列表接口需要分页参数：`page`、`page_size`；当前开发期小数据列表可先返回完整数组，但接口文档必须注明用途和后续分页策略。
- 创建接口返回创建后的资源或任务 ID。
- 耗时操作返回任务对象，不在 HTTP 请求里阻塞等待执行结束。
- 错误响应统一包含：

```json
{
  "code": "PARSE_JOB_NOT_FOUND",
  "message": "解析任务不存在",
  "detail": {}
}
```

## 数据库规范

当前开发期：

- 当前开发期不要直接引入真实数据库；根目录 `data/*.json` 是临时持久化表。
- JSON 表名应与业务集合一致，例如 `projects.json`、`fields_by_file.json`、`report_versions.json`。
- 写接口必须调用 store 的保存逻辑，避免只改内存状态。
- 需要追溯的集合必须单独保留记录，例如 `operation_logs.json`、`report_versions.json`、`rule_template_versions.json`、`deleted_projects.json`。

后续接入 PostgreSQL 时：

- 所有核心表必须包含 `id`、`created_at`、`updated_at`。
- 审计敏感表应包含 `created_by`、`updated_by`。
- 业务删除优先软删除，除非明确允许物理删除。
- 规则模板和报告草稿必须保留版本信息。
- 解析结果必须保存来源文件、页码或定位信息，支持后续追溯。

## 权限规范

- API 默认需要认证，登录、健康检查除外。
- 用户、规则、日志等管理接口需要管理员权限。
- 编制员只能操作有权限的项目和报告。
- 审核员可查看报告和溯源信息，但默认不能修改规则。

## 日志规范

- 用户关键操作写操作日志。
- 后台任务执行过程写解析日志或任务日志。
- 异常日志必须包含 request id、user id、module、action 和 error code。
- 日志中不得记录明文密码、完整访问令牌、对象存储密钥和完整模型密钥。

## AI 边界规范

- Core API 不拼接复杂 Prompt。
- Core API 不依赖具体模型供应商。
- Core API 和 AI Worker 通过明确 schema 交换任务和结果。
- AI Worker 返回的原始模型响应可以归档，但前端默认只展示摘要、结构化字段和必要溯源。

## 测试规范

- 新增 API 至少补充成功路径测试。
- 涉及权限的接口需要覆盖未登录、无权限和有权限三类场景。
- 解析任务编排需要覆盖任务创建、状态流转、失败重试。
- 不要求单元测试直接调用真实大模型。
