# 后端需求说明

## 背景

后端 Core API 服务承接智能检测报告生成系统的基础 Web 能力和业务编排能力。当前阶段已经为前端提供可联调接口，并以仓库根目录 `data/*.json` 作为开发期轻量数据层；OCR、文档解析、字段抽取和大模型推理暂不在 Web API 中真实执行，后续通过任务队列调度 AI Worker。

## 架构定位

Core API 是系统主业务后端，负责：

- 登录认证、用户、用户偏好、角色和权限边界。
- 项目、文件、字段、规则、报告、消息和日志等业务数据。
- 解析状态、解析事件、字段集合、报告版本和交付记录的维护。
- gen-report 工作流编排：manifest 构建、运行管理、审批、字段设置和报告生成。
- 后续与对象存储、数据库、任务队列和 AI Worker 的集成。

AI Worker 是独立执行层，负责：

- OCR、PDF、图片、Word、Excel 等文件解析。
- 调用大模型完成字段抽取、归一化、补全和校验。
- 产出结构化字段、置信度、来源位置、解析日志和错误信息。

前端不直接调用 AI Worker。前端只调用 Core API，由 Core API 创建任务、查询状态并聚合结果。

## 用户与角色

| 角色 | 主要能力 |
| --- | --- |
| 管理员 | 管理用户、规则模板、项目、日志和异常任务 |
| 编制员/检测人员 | 上传原始记录、发起解析、查看结果、生成和编辑报告 |
| 审核员 | 查看报告、字段溯源、规则版本和操作记录 |

## 核心模块

### 认证与用户管理

- 用户名密码登录。
- 本地 JWT 访问令牌签发、会话校验和登出。
- 用户偏好读取和更新，用于保存当前项目等工作台状态。
- 用户新增、编辑、启用、禁用、批量导入和导出。
- 角色和权限控制。
- 登录成功和失败均写入审计日志。

### 项目管理

- 项目列表、详情、状态和进度。
- 项目下文件、解析任务、报告草稿的聚合视图。
- 项目状态由业务事件驱动更新，例如解析中、待生成、待审核、待上传、已完成。
- 当前支持新建、软删除、查看已删除项目和恢复项目。

### 文件管理

- 原始文件上传元数据登记。
- 文件类型、大小、重复文件、空文件和权限校验。
- 当前文件元数据写入 `data/raw_files.json`，文件级解析事件写入 `data/parse_events_by_file.json`，字段写入 `data/fields_by_file.json`。
- 后续文件内容写入 MinIO/S3，数据库只保存对象 key 和元数据。
- 支持文件预览、删除、重命名和排序。

### 解析任务管理

- 基于项目、文件列表、检测类型和规则模板创建解析任务。
- 当前通过 `PATCH /records/files/{file_id}/status` 维护文件级解析状态：解析中、解析成功、解析失败。
- 后续任务进入队列后由 AI Worker 消费，并维护任务状态：待处理、运行中、成功、失败、已取消。
- 记录阶段进度：上传文件、OCR识别、文档结构解析、大模型提取、结构化存储。
- 支持失败重试和错误原因查询。

### 规则配置

- 模板分类树、模板详情、字段定义和规则配置。
- 模板新建、复制、编辑、字段编辑、保存规则和版本历史。
- 后续补齐模板版本发布、归档和回滚。
- 报告生成和解析任务必须记录使用的模板 ID 和版本号。
- 规则调用可追溯。

### 报告生成与编辑

- 基于解析结果和规则模板生成报告草稿。
- 支持章节目录、正文内容、章节类别、目录排序、保存草稿、版本记录、预览记录、导出记录、更正版上传和版本回退。
- 当前导出 Word/PDF 先记录交付文件名和任务状态，后续接入真实模板渲染和对象存储下载。
- 当前系统不包含报告审核流程；报告生成后提供预览、导出、更正版上传和版本管理能力。

### gen-report 工作流

- gen-report 工作流是 Core API 内置的报告生成管线，包含四个阶段：validate → prepare → extract → generate。
- `manifest_builder.py` 根据项目规则模板、源文件和元数据生成 manifest.yaml 和工作空间目录。
- `gen_report_service.py` 编排工作流运行、审批、字段设置和报告生成，维护 run 级状态和工作空间目录。
- 前端通过 `/api/v1/gen-report/*` 端点调用工作流：创建运行、查询进度、审批、设置字段、生成报告。
- 工作流支持运行级状态恢复（重启后从工作空间目录的 status.json 恢复）。
- 外部 Python 包 `gen-report` 提供 validate/prepare/extract/generate 四阶段的实际执行能力。

### 日志与消息

- 操作日志：用户关键操作、对象、结果、时间、IP 和用户代理。
- 系统日志：服务运行异常、集成错误、后台任务错误。
- 解析日志：Prompt 摘要、模型返回摘要、结构化 JSON、用户修正历史。
- 消息通知：解析完成、解析失败、报告提交、审核反馈等。

## 接口边界

对前端开放的接口使用 `/api/v1` 前缀：

```text
/api/v1/auth/*
/api/v1/messages*
/api/v1/projects/*
/api/v1/records/*
/api/v1/rules/*
/api/v1/reports/*
/api/v1/gen-report/*
/api/v1/system/*
```

内部 AI 能力不直接暴露给前端。优先通过队列传递任务；如果后期需要 HTTP 内部接口，使用 `/internal/*` 前缀，并通过服务间密钥、内网访问控制和审计日志保护。

## 非目标

- Core API 不直接实现大模型 Prompt 细节。
- Core API 不直接保存原始大文件内容到数据库。
- 前端不绕过 Core API 直接访问对象存储或 AI Worker。
- 当前阶段不做完整微服务治理、服务注册和分布式事务。
